# Simplex AI - Project Specifications (V1)

## 1. General Vision
Simplex AI is an assistant-type AI agent, optimized for office work environments (corporate/administrative). Although its "brain" is completely separate, it will be distributed with a simple and friendly graphical interface, hiding technical complexity from the end user.

## 2. Core Architecture
*   **Paradigm:** Developed from scratch, total control over the agent loop.
*   **LLM Backend:** Agnostic, using the `litellm` library with `acompletion(stream=True)` for async streaming. Allows easy switching between models (OpenAI, Anthropic, Gemini, Ollama for local execution).
*   **Async-Native:** The entire stack is async — Flet UI runs on an async event loop, LLM calls are non-blocking, and streaming responses update the UI chunk-by-chunk in-place.
*   **Configuration:** `pydantic-settings` via `.env` file (model, API keys, temperature, max_tokens, system prompt).
*   **Python Ecosystem:** 
    *   Language: Python
    *   Package Manager: `uv` (for speed and efficient environment management).

## 3. Extensibility System
*   **Sub-agents:** Delegation system to specialized agents with their own prompts and tools. [IMPLEMENTING]
    *   *Architecture:* Isolated LLM calls with specific system prompts for internal tasks (e.g., reranking, analysis).
    *   *Integration:* Used within native tools to process high-volume data before returning to the main agent.
*   **Tools (Native tools):** Python functions decorated to be exposed to the LLM.
*   **MCP Integration (Model Context Protocol):** Integrated client to connect to MCP servers (for standardized access to file system, databases, external applications).

## 4. User Interface (UI)
*   **Framework:** `NiceGUI` (High-level UI components based on Vue and Quasar, written entirely in Python).
*   **Architecture:**
    *   Entry: `ui.run()` — NiceGUI auto-detects and runs the async event loop.
    *   UI components: `App` (top-level layout) → `ChatView` (message list + input bar).
    *   No CLI — the NiceGUI app is the only user-facing interface.
*   **Streaming:** Assistant messages stream token-by-token into a `ui.markdown` widget. Each chunk from `litellm.acompletion(stream=True)` appends to the bubble content and triggers a UI update. No threads, no blocking.
*   **Experience:** Chat style (bubble messages), clean, intuitive for non-technical users. User bubbles right-aligned, assistant left-aligned.
*   **NiceGUI Advantages:** 
    *   Native Tailwind CSS support for easy and modern styling.
    *   Standard Web technologies (HTML/JS/CSS) under the hood, but 100% Python development.
    *   Extremely flexible layout system.
    *   Easy integration with existing web tools and icons (Material Design, Font Awesome).

## 5. Key Features (Office Focus)
*   **Document & RAG Features (Core System):**
    *   **Drop & Chat (Local Injection):** Drag and drop files directly into the UI. The agent reads the document on the spot, vectorizes it (if necessary), and answers questions.
    *   **Multi-Format Support:** Text extraction from `.pdf`, `.docx`, `.xlsx` (tabular data), and `.pptx` using dedicated packages (e.g., `unstructured`, `pandas`).
    *   **Citations (Source of Truth):** Any response generated based on a document will contain explicit references (e.g., *"According to contract_v2.pdf, page 3 [Link]"*).
*   **Office Automations (via Tools & Skills):**
    *   **Email Drafter / Reader:** Connection to Outlook/Gmail (IMAP/MS Graph) to read recent emails and draft context-based responses (without automatic sending).
    *   **Calendar & Scheduling:** Check agenda and generate `.ics` files for quick calendar addition.
    *   **Data Analysis Sub-agent:** Dedicated agent for analyzing Excel/CSV files by executing Python code (Pandas) in the background, optimal for tabular data compared to standard RAG.
*   **File System & Organization (via MCP):**
    *   **Local File Search:** Quick search on the user's machine to find specific documents.
    *   **File Renamer & Organizer:** Bulk logical sorting and renaming of files (e.g., scanned documents renamed based on content - `Invoice_X_Month_Year.pdf`).

## 6. Usability & Accessibility (Templates & Quick Actions)
*   **Predefined Prompts (Quick buttons in UI):**
    *   **"One-Click" Summarization:** Generate an executive summary for any loaded document.
    *   **Grammar Check:** Adjust raw text for spelling and fluency.
    *   **Translate to English (Formal Tone):** Translation adapted for business correspondence.
    *   **Extract Action Items:** Identify key "to-do" points from an email or document.
    *   **Convert to Bullet-points:** Quick formatting of long texts into easy-to-read lists.
*   **Easy Export:** Button to quickly save the response generated by the agent in `.docx` (Word) or `.pdf` format on the user's desktop.

## 7. Security & Privacy
*   **"Confidential" Mode (Local Execution):** Use of a locally run model (via Ollama) to analyze documents containing sensitive data (SSNs, financial details, confidential clauses).
*   **Data Redaction:** Intermediary tool that automatically anonymizes names and contact details before sending the information to an external API (OpenAI/Anthropic/Google).

## 8. Proposed Development Phases
1.  **Core Engine, LLM & Flet UI:** Codebase with `uv`, `litellm` integration (async streaming), history management, and the Flet chat UI. Merged from original phases 1 and 3 — no CLI, UI from day one.
2.  **Tool System & MCP:** Creating the extensibility architecture (decorators for tools, modules for skills) and the MCP client.
3.  **Office Skills & UI Templates:** Adding quick buttons (Summarize, Correct, etc.), the Data Analysis sub-agent, and Export functionalities.
4.  **Security & Packaging:** Adding simplified support for local models (Ollama) and creating the build process for `.exe`/Linux binary.
